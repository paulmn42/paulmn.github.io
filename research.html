Summary of Findings

The first test I did was the competitor’s analysis and it provided a base line for my entire project. The thing I learned from it was the more basic features. All the competitors had networking in some ways so contact information was the natural thing to incorporate. Something that all of them lack was the amount of customization, so I focused on really expressing myself, to the extent of my lack programing could afford. Next was the user interviews which gave me structure to the pages and told me which pages to implement as well as giving me clues to how to customize my web pages. Lastly was the feature matrix and prioritization which help me, well prioritize what features to add to my website

Next would be the users testing. User Stories remined me that I forgot to implement skills which was one the first things I fixed. The next thing I did was the usability hub testing, while the first the click test just wasn’t too helpful as people mostly clicked on the right thing the question test did reveal that I need to be clearer with my job description. Finally, would be my cognitive walkthrough. For the walkthrough with a student of my projects page need more structure and description. Next, for the walkthrough with the TA told me to take a professional head shot, and fix my contact page. Lastly, my sister pointed out grammatical errors and to elaborate on my about page.
